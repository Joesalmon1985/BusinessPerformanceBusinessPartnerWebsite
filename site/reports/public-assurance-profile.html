KO41a, ERIC, DSPT and regulatory context — public data only

Agent-assisted analytical brief — public-data demonstration: This is not an official Dorset HealthCare report. It uses public aggregate data only and requires human review and local owner confirmation before operational use.

What this report demonstrates

Bringing together KO41a, ERIC, DSPT and CQC context into an assurance briefing with per-source commentary, appropriate-use language, and DSPT history — not a clinical performance scorecard.

The question given to the agent

Which public assurance artefacts contain RDY rows, what is each useful for, what are the gaps (FFT org-level, CQC non-statistical context), and how should a Business & Performance Partner use this as a conversation starter?

Prompt excerpt

Index public assurance sources for RDY: KO41a, ERIC, DSPT, FFT, CQC.
One commentary card per theme with appropriate-use language.
Summarise DSPT history descriptively if multiple rows exist.
No operational IG or complaints performance conclusions.

Data used

  • demo_assurance_profile.csv
  • KO41a, ERIC, DSPT processed extracts
  • CQC context note (regulatory background only)

KO41a 2024-25; ERIC 2024/25; DSPT public history; FFT/CQC as noted.

RDY filter: ODS code RDY and case-insensitive trust name variants (Dorset HealthCare / Dorset Healthcare University NHS Foundation Trust). See site/public-data/metadata/filter_note_*.txt for per-source notes.

Agent process demonstrated

  1. Check DATA_SOURCE_REGISTER for assurance-related sources
  2. Confirm RDY row presence in KO41a, ERIC and DSPT extracts
  3. Record FFT download outcome (no org-level RDY rows in summary XLSX)
  4. Capture CQC provider page as context note only
  5. Draft per-theme commentary with agent flags and appropriate use
  6. Summarise DSPT assessment history descriptively
  7. Link demo CSV, filter notes and governance documentation

How to read this report

  • Each theme row indexes a public assurance source — not a combined performance score.
  • KO41a and ERIC are typically annual returns; DSPT is an annual IG assessment history.
  • FFT org-level gap means patient experience cannot be summarised from the downloaded summary file alone.
  • CQC content is regulatory context — not statistical performance data.
  • Use this brief to start assurance conversations with named return owners — not as an official Trust assurance report.
  • Demonstration only — not official Dorset HealthCare reporting.

Public open data confirms RDY participation in annual written complaints (KO41a) and ERIC estates returns, with a multi-year DSPT public assessment history.

Latest DSPT public assessment: 2024-25 (version 7) - Standards met. History shows 16 published rows — movement from earlier 'Approaching standards' to recent 'Standards met' is descriptive assurance context, not proof of operational IG detail.

FFT org-level rows were absent in the downloaded summary file; CQC provides regulatory context only. A business partner would route operational interpretation to named return owners before any assurance use.

Agent-generated observations

  • Public KO41a row confirms presence — local complaints team owns interpretation.
  • ERIC row confirms estates return participation — check amendments file.
  • DSPT: 16 history rows — annual assurance signal only.
  • FFT summary lacked RDY org rows — setting-level XLSX may be needed.
  • CQC page is regulatory background — not performance proof.

What cannot be concluded from this data

  • Operational IG compliance from DSPT status alone
  • Patient experience levels from missing FFT org-level data
  • Regulatory rating or inspection outcome as a performance proxy
  • Complaints volume trends from a single annual KO41a snapshot
  • Estates efficiency or cost performance from ERIC row presence alone

Questions for a Business & Performance Partner

  • Is this the same definition used locally?

    Public datasets may use national definitions. Local dashboards may use different filters, denominators or reporting dates.

  • Is this still the latest position?

    Public data may be a delayed publication. Local operational data may have moved on since the reporting period shown.

  • Who owns the narrative?

    The accountable service, finance, workforce, quality or BI owner should confirm the explanation before the finding is used.

  • Is this a performance issue, a data quality issue, or a service model issue?

    A public figure can be affected by coding, exclusions, small numbers, local pathways or publication rules.

  • Who is the named owner for each return this cycle?

    KO41a, ERIC, DSPT and FFT each have accountable teams — confirm before citing in assurance packs.

  • Is manual FFT setting-level download needed to fill the org-level gap?

    The summary XLSX lacked RDY org rows — patient experience lead should confirm next step.

  • Have ERIC amendments been published post-release?

    Estates returns may revise — check amendments file before board use.

How to verify these figures

  1. Open the linked demo CSV or audit file for the figure you are checking.
  2. Locate the RDY row using the filter notes and ODS code RDY.
  3. Confirm the value matches the published source — this demo does not recalculate NHS England comparators.
  4. Check metric definitions, provisional/final status and suppression before any operational use.
