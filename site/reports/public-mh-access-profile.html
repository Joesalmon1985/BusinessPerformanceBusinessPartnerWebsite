Worked example: AI-assisted MHSDS public-data briefing

MHSDS — RDY provider measures (provisional public data)

Agent-assisted analytical brief — public-data demonstration: This is not an official Dorset HealthCare report. It uses public aggregate data only and requires human review and local owner confirmation before operational use.

What this report demonstrates

Summarising mental health access and activity from public MHSDS data with plain-English measure commentary, agent flags, and descriptive monthly trends where a downloaded time-series extract exists — not an operational access dashboard.

The question given to the agent

From public MHSDS data for RDY, explain what key access and activity measures mean, what the agent noticed (including suppression and breakdown limits), where month-on-month trends are supported by the downloaded time series, and what a mental health data owner must confirm before use.

Prompt excerpt

Filter provider RDY from MHSDS monthly CSV.
Add plain-English commentary for MHS23, MHS01, MHS29, MHS69, CYP32a and suppression.
Compute trends only from downloaded time-series file (Provider breakdown).
Do not sum incompatible breakdowns. No causal claims.

Data used

  • MHSDS Monthly Statistics (demo_mhsds_activity.csv — latest month slice)
  • MHSDS time series (rdy_mhsds_monthly_mhsds_monthly_performance-april-2026_time_series_MHSDS Time_Series_data_Apr2025_Mar_Perf_2026_v2.csv — Provider RDY, Apr 2025–Mar 2026)
  • trend_mhs23_rdy.csv — MHS23 historic stack (8 months)

01/04/2026 to 30/04/2026

RDY filter: ODS code RDY and case-insensitive trust name variants (Dorset HealthCare / Dorset Healthcare University NHS Foundation Trust). See site/public-data/metadata/filter_note_*.txt for per-source notes.

Agent process demonstrated

  1. Open MHSDS demo CSV and Provider RDY time-series extract from processed/
  2. Count numeric vs suppressed values; select MHS23, MHS01, MHS29, MHS69, CYP32a as key figures
  3. Load trend_mhs23_rdy.csv for MHS23 where not in time-series bundle
  4. Build Key figures explained with previous-period comparators where ≥2 months exist
  5. Draft commentary cards and descriptive trend charts — no causal claims
  6. Filter to RDY using ODS code and Dorset HealthCare trust name variants (see filter notes)
  7. Detect current vs historic periods from reporting columns or stacked trend files
  8. Select key figures from named measures, top-N tables or source-presence checks — not exhaustive lists
  9. Decide trend analysis only when ≥2 comparable periods exist and values are not suppressed
  10. Avoid overclaiming — no causal language, no fabricated targets, no recalculated peer comparators
  11. Document verification paths and what a human reviewer must still confirm

How to read this report

  • Each row is one MHSDS measure for RDY with a specific breakdown (Provider, ICB-resident, referral source, etc.).
  • MEASURE_VALUE is the published count or rate for 01/04/2026 to 30/04/2026. '*' means suppressed (small numbers).
  • Provider rows (PRIMARY_LEVEL=RDY, BREAKDOWN=Provider) are trust-as-provider counts; ICB-resident rows describe people registered in an ICB area.
  • You cannot sum rows across different breakdowns to produce a single trust-wide KPI.
  • Monthly MHSDS is typically provisional until end-of-year refresh — confirm publication status before use.
  • This brief is a worked example using public aggregate data — not official Dorset HealthCare reporting.

First-draft analysis

For 01/04/2026 to 30/04/2026, the public MHSDS extract contains 200 RDY measure rows (123 numeric, 77 suppressed). Provider-level open referrals (MHS23) show 24,790 — a stock measure the agent would route to the MHSDS owner before any performance use.

The downloaded time series (11 Provider months for MHS01) shows in-contact counts rising from 21,445 (Feb 2026) to 21,755 (Mar 2026) — descriptive only; the agent would not infer cause without local pathway context.

Referral-related numeric rows sum to 157,235 across breakdowns — not a single headline referral count. This first draft supports scoping questions for local MHSDS validation, not directorate performance conclusions.

Agent-generated observations

  • Provider-level rows exist but ICB-resident and referral-source breakdowns also appear — not all rows are trust-wide totals.
  • Referral-related numeric rows sum to 157,235 across breakdowns — not a single headline referral count.
  • MHS01 in-contact trend available across 11 months in time-series extract.
  • MHS23 open referrals trend available across 8 months via trend_mhs23_rdy.csv.
  • Monthly MHSDS data are typically provisional until end-of-year refresh.

What cannot be concluded from this data

  • Operational cause of access or caseload changes without local pathway and PAS context
  • A single trust-wide referral or caseload total from summed breakdown rows
  • ICB-resident vs provider comparability without local definition checks
  • Clinical quality or pathway performance from activity counts alone
  • That month-on-month movement in MHS01/MHS29 implies demand, capacity or performance without owner confirmation

Questions for a Business & Performance Partner

  • Is this the same definition used locally?

    Public datasets may use national definitions. Local dashboards may use different filters, denominators or reporting dates.

  • Is this still the latest position?

    Public data may be a delayed publication. Local operational data may have moved on since the reporting period shown.

  • Who owns the narrative?

    The accountable service, finance, workforce, quality or BI owner should confirm the explanation before the finding is used.

  • Is this a performance issue, a data quality issue, or a service model issue?

    A public figure can be affected by coding, exclusions, small numbers, local pathways or publication rules.

  • Which MHSDS measures map to local directorate access KPIs?

    Public measure IDs may not match local dashboard names — confirm mapping with the MHSDS/data owner.

  • Are suppressed cells material for Dorset ICB (11J) residents?

    77 suppressed values in this extract — check whether key access measures are affected for your ICB scope.

How to verify these figures

  1. Open the linked demo CSV or audit file for the figure you are checking.
  2. Locate the RDY row using the filter notes and ODS code RDY.
  3. Confirm the value matches the published source — this demo does not recalculate NHS England comparators.
  4. Check metric definitions, provisional/final status and suppression before any operational use.
