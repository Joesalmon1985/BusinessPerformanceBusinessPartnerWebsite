CSDS — RDY community activity (March 2026)

Agent-assisted analytical brief — public-data demonstration: This is not an official Dorset HealthCare report. It uses public aggregate data only and requires human review and local owner confirmation before operational use.

What this report demonstrates

Turning public CSDS community services data into a structured briefing with plain-English activity commentary, descriptive historic trends where the stacked extract supports them, and agent flags — not a directorate performance dashboard.

The question given to the agent

Using public CSDS for March 2026, explain community activity measures in plain English, show descriptive month-on-month trends where the historic stack supports them, and what a business partner should validate locally.

Prompt excerpt

Filter CSDS monthly CSV to RDY. Inspect CareActivities by ActivityType.
Add Key figures explained and commentary for assessment, clinical intervention and limits.
Use trend_csds_activity_rdy.csv for month-on-month trends where ≥2 periods exist.
No causal language.

Data used

  • CSDS Monthly Statistics (demo_csds_activity.csv — March 2026)
  • trend_csds_activity_rdy.csv — historic CareActivities stack (8 months)

2026-03-01 to 2026-03-31

RDY filter: ODS code RDY and case-insensitive trust name variants (Dorset HealthCare / Dorset Healthcare University NHS Foundation Trust). See site/public-data/metadata/filter_note_*.txt for per-source notes.

Agent process demonstrated

  1. Open demo_csds_activity.csv and trend_csds_activity_rdy.csv from processed/
  2. Filter to CareActivities / ActivityType dimension for RDY
  3. Build Key figures explained with previous-period comparators for Assessment and Clinical Intervention
  4. Render descriptive trend charts when historic stack has ≥2 comparable months
  5. Filter to RDY using ODS code and Dorset HealthCare trust name variants (see filter notes)
  6. Detect current vs historic periods from reporting columns or stacked trend files
  7. Select key figures from named measures, top-N tables or source-presence checks — not exhaustive lists
  8. Decide trend analysis only when ≥2 comparable periods exist and values are not suppressed
  9. Avoid overclaiming — no causal language, no fabricated targets, no recalculated peer comparators
  10. Document verification paths and what a human reviewer must still confirm

How to read this report

  • Each row is a CSDS measure for RDY as provider for March 2026 in the demo extract.
  • Assessment and Clinical Intervention trends use trend_csds_activity_rdy.csv (8 monthly periods, Aug 2025–Mar 2026).
  • This report filters to DIMENSION=ActivityType and MEASURE=CareActivities — other CSDS dimensions are not shown.
  • MEASURE_DESC describes the activity type (e.g. Assessment, Clinical Intervention); MEASURE_VALUE is the count.
  • Age-band columns split the same count where published — they are not additional unique patients.
  • Public aggregate CSDS cannot prove referral demand, waiting times or team performance without local context.
  • This brief is a worked example — not official Dorset HealthCare reporting.

First-draft analysis

For 2026-03-01 to 2026-03-31, the public CSDS extract contains 200 RDY rows. Care activity totals in the ActivityType slice sum to 23,970.

Assessment (4,270) and clinical intervention (5,955) dominate numeric activity — the agent would ask the community services owner to confirm coding and service scope before any review use.

Historic CSDS stack (trend_csds_activity_rdy.csv (historic stack from script 05)) supports descriptive month-on-month comparison for key activity types — not causal operational conclusions.

Agent-generated observations

  • Clinical intervention and assessment dominate care activities where numeric values exist.
  • Health visitor activity may appear in CYP age bands — confirm service scope locally.
  • Historic monthly stack available (trend_csds_activity_rdy.csv (historic stack from script 05)) — demo month March 2026 plus 8 trend periods.
  • Public aggregate CSDS does not support team-level conclusions.

What cannot be concluded from this data

  • Activity trends or period-on-period change from one month
  • Directorate-level performance or operational priorities
  • Coding quality or submission completeness
  • Referral demand from activity counts alone
  • Team or pathway performance from public aggregate rows

Questions for a Business & Performance Partner

  • Is this the same definition used locally?

    Public datasets may use national definitions. Local dashboards may use different filters, denominators or reporting dates.

  • Is this still the latest position?

    Public data may be a delayed publication. Local operational data may have moved on since the reporting period shown.

  • Who owns the narrative?

    The accountable service, finance, workforce, quality or BI owner should confirm the explanation before the finding is used.

  • Is this a performance issue, a data quality issue, or a service model issue?

    A public figure can be affected by coding, exclusions, small numbers, local pathways or publication rules.

  • Which CSDS measures feed local community dashboards?

    Public measure codes may differ from local dashboard labels — confirm mapping with the CSDS owner.

  • What was coding QA status for March 2026?

    Activity counts depend on timely, accurate CSDS submissions — check DQ reports before using figures.

  • Does the full RDY processed extract contain activity rows not in the demo slice?

    The demo CSV is a convenience sample — verify against the full processed file if figures look sparse.

How to verify these figures

  1. Open the linked demo CSV or audit file for the figure you are checking.
  2. Locate the RDY row using the filter notes and ODS code RDY.
  3. Confirm the value matches the published source — this demo does not recalculate NHS England comparators.
  4. Check metric definitions, provisional/final status and suppression before any operational use.
