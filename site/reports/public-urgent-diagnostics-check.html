Urgent care, diagnostics and beds — public source applicability check (RDY)

Public-data demonstration report: This is not an official Dorset HealthCare report. It uses public aggregate data only and requires human review and local owner confirmation before operational use.

What the agent was asked to do

Business question: Check whether RDY appears in public A&E, DM01 and KH03 files; explain what each source can safely support for a trust without an emergency department; and where trend analysis is or is not available.

Dataset: A&E, DM01 and KH03 public provider files (RDY)

This is a first-draft analytical brief for human review — not an approved performance report.

Data used

  • A&E monthly provider CSV (rdy_ae_monthly_ae_monthly__May-2026-CSV-F4flrg.csv)
  • DM01 (demo_dm01_diagnostics.csv + full RDY extract)
  • KH03 (demo_kh03_beds.csv + trend file where used)
  • trend_ae_rdy.csv
  • trend_dm01_rdy.csv
  • Period covered: A&E Dec 2025 – May 2026; DM01 Nov 2025 – Apr 2026; KH03 quarterly snapshots (Mar 2024 to Jun 2024).
  • RDY filter: ODS code RDY; Dorset HealthCare trust name variants (see filter notes)
  • Historic trend data: Yes — six-month A&E/DM01 stacks where available; KH03 quarterly snapshots (KH03 supporting extract: 6 quarterly snapshots from 2023-03-31 to Jun 2024 (may lag monthly A&E/DM01 dates).)

What this report can and cannot tell us

This report can:

  • Confirm RDY row presence in public A&E, DM01 and KH03 files.
  • Show descriptive trends where stacked historic files support them.
  • Explain what each source can safely support for a trust without a Type 1/2 emergency department.

This report cannot:

  • Prove urgent care or diagnostic performance standing from source validation alone.
  • Make ED performance claims — RDY shows zero Type 1/2 A&E attendances by service model.

For that reason, this should be treated as a first-draft prompt for review, not a final performance judgement.

Why this is useful

  • The agent checked RDY presence across heterogeneous urgent-care, diagnostics and bed-stock sources.
  • It explained which sources are meaningful for a mental health/community trust and where local owners must confirm definitions.

Human validation checklist

  • Is this the same definition used locally? — Public datasets may use national definitions. Local dashboards may use different filters, denominators or reporting dates.
  • Is this still the latest position? — Public data may be delayed. Local operational data may have moved on since the reporting period shown.
  • Is the figure affected by data quality, suppression, coding or service model? — A public figure can be affected by small numbers, exclusions, coding rules or how services are configured.
  • Who owns the local narrative? — The accountable service, finance, workforce, quality or BI owner should confirm the explanation before use.
  • Is this suitable for board/service reporting, or only a prompt for further review? — These briefs are first drafts — confirm whether the figure is ready for formal reporting or needs more local work.
  • Which urgent care metrics apply locally without an ED? — Many national urgent care indicators assume Type 1/2 ED activity — confirm which apply to RDY.

Bottom line

This is a source applicability check, not a unified performance report. A&E rows validate RDY's service model rather than ED performance. DM01 mainly describes audiology/community diagnostics waiting lists for RDY. KH03 provides quarterly mental health bed-stock context. Local owners should confirm all figures before operational use.
