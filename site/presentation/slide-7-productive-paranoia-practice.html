Speaker notes

This slide is about not letting performance narrative run ahead of assurance. Must-not-fail measures with confidence tiers. When a dashboard refreshes, ask whether the definition underneath changed — reconcile before explaining variances to services or the board.

Epic and ADF produce quiet failures: pipeline success while an event type stopped flowing; workflow change leaving a field null that downstream logic depends on. The reporting assurance deliverable on the site combines risk, validation and confidence for exactly this transition period.

Senior brief example shows how to communicate with caveats — performance insight with explicit confidence, not false precision.

What this slide delivers — plain English

Must-not-fail measures + confidence tiers
Critical reports (board, NHSE, contract) flagged with High / Medium / Low / Do not use ratings. B&P maintains the register and enforces “no Board headline when confidence is Low.”
Reporting assurance during migration
Living pack of risks, scheduled checks, reconciliation results and publish decisions for the transition period. B&P is accountable for keeping it current as Epic and ADF phases progress.
Data quality & confidence report
Formatted view of the confidence register for BI and performance leads — which products are safe for service review vs need owner confirmation vs must not reach the Board.
Reconciliation (definition migration)
Monthly compare: old system count vs new system count vs agreed figure for the same cohort. B&P narrates the gap and prevents false “improvement” or “crisis” stories.
Low-confidence preset (source-to-report map)
Filtered view of lineage rows where confidence is Low or broken — used to require owner sign-off before figures enter directorate or board packs.
Senior performance brief
Monthly narrative template for SLT/Board with a confidence column per section. B&P drafts honest headlines (“dashboard says X; agreed figure is Y until reconciliation completes”).
Quiet failures (Epic / ADF)
Examples: pipeline green but referral event missing; workflow change leaves triage field null. B&P assurance catches these before they become management decisions.

Suggested visual: traffic-light confidence strip; “pipeline green / event missing” callout for ADF; reconciliation screenshot optional.
