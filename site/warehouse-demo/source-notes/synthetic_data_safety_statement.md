# Synthetic data safety statement

## Demonstration only

All files under `source-data/` are **entirely synthetic**. They were generated by [`create_synthetic_source_data.py`](../generator/create_synthetic_source_data.py) using a fixed random seed. They must not be used for:

- Operational reporting or performance management
- Patient care or clinical decisions
- IG training involving real identifiers
- Implying real Dorset HealthCare or NHS provider activity

## No real data

This pack does **not** contain:

- Real patient data or real NHS numbers
- Real staff identities or real usernames
- Real supplier schemas or confidential internal definitions
- Reverse-engineered public RDY / MHSDS figures

## Identifier conventions

| Identifier type | Format | Notes |
|-----------------|--------|-------|
| Patient pseudo ID | `DRH-PAT-000001` | Internal demo key |
| NHS number (demo) | `DEMO-NHS-000001` | **Not a valid NHS number** — must not pass NHS number validation |
| CareCall contact | `CC-YYYYMM-#####` | |
| CareCase case | `CS-#######` | |
| Staff | `DRH-STF-####` | |

> **These are not valid NHS numbers and must not pass NHS number validation.**

## Fictional systems

CareCall, CareCase, Legendary Care, RosterFlow and LedgerWise are **invented system names**. Field names and file layouts are plausible for demonstration but do not reproduce any live Trust or supplier extract.

## Regeneration

To reproduce the pack:

```bash
cd site/warehouse-demo/generator
python create_synthetic_source_data.py
```

## Human sign-off

Any future report or warehouse design produced from this demo requires human validation before it could be used outside this demonstration site.
